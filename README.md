## Описание проекта

Проект решает задачу автоматизации продаж в мессенджере Telegram. Клиенты совершают покупки чеез нативный интерфейс Mini App, что обеспечивает высокую конверсию за счет необходимости скачивать отдельное приложение.

Административная часть построена по классической MVC-архитектуре с использованием Thymeleaf. Доступ к ней защищён стандартной сессионной авторизацией Spring Security с BCrypt-паролями, а данные Telegram Mini App проверяются на сервере с использованием HMAC-SHA256.

## Ключевые возможности

### Клиентская часть (Telegram Mini App)
*   **Бесщовная авторизация:** автоматическая идентификация пользователя через `initData` с криптографической проверкой подписи бота.
*   **Управление корзиной:** Работа с сессиями без обязательной предварительной регистрации.
*   **Программа лояльности:** Динамический расчет персональной скидки на основе исторической суммы покупок (`total_spent`).
*   **История заказов:** Доступ к архиву только подтвержденных администратором транзакций.

### Административная панель (Web Interface)
*   **Управление каталогои:** Полный цикл CRUD для товаров, категорий, брендов и вариантов (вкусов/цветов). Поддержка загрузки изображений с валидацией MIME-типов.
*   **Обработка заказов:** Изменение статусов заказов. Начисление баллов лояльности происходит строго при переходе в статус `COMPLETED`.
*   **Система уведомлений:** Мгновенные оповещения о новых заказах в Telegram со ссылкой на защищённый список заказов.
*   **Управление клиентами:** Просмотр базы клиентов, их суммарных трат и текущих уровней скидок.

## Архитектурные решения

Проект построен с соблюдением принципов коммерческой разработки:

1.   **Слоистая архитектура:** Строгое разделение на Controller (HTTP), Service (бизнес-логика) и Repository (доступ к данным).
2.   **Безопасность данных:**
     *   Валидация `initData` от Telegram на бэкенде через HMAC-SHA256 для защиты от подмены `userId`.
     *   Сессионная авторизация администраторов через Spring Security и хранение паролей в виде BCrypt-хешей.
3.   **Целостность транзакций:** Логика начисления баллов лояльности инкапсулирована в сервисе заказов и выполняется только при подтверждении оплаты/выдачи товара.
4.   **Оптимизация запросов:** Использование `JOIN FETCH` в JPA для предотвращения проблемы N+1 при загрузке заказов с позициями.
5.   **Контейнеризация:** Полная изоляция приложения и базы данных через Docker Compose с настройкой health-checks.

## Быстрый старт

Для локального запуска проекта вам потребуется установленный Docker и Docker Compose.

### 1. Конфигурация окружения
Создайте файл `.env` в корне проекта на основе примера:

```bash
cp .env.example .env
```

Заполните необходимые переменные:
- `DB_PASSWORD` - пароль для PostgreSQL.
- `TELEGRAM_BOT_TOKEN` - токен ващего бота от @BotFather
- `TELEGRAM_ADMIN_CHAT_ID` - ваш числовой ID в Telegram
- `APP_BASE_URL` - публичный адрес приложения (например, через ngrok или домен).

### 2. Запуск контейнеров

Выполните команду в терминале:

```bash
docker-compose up --build -d
```

### 3. Доступ к приложению

- **Веб-сайт / Mini App:** http://localhost:8080
- **Админ-панель:** http://localhost:8080/admin

**Данные для первого входа:**
Логин и пароль генерируются автоматически при первом запуске и выводятся в логи контейнера (`docker logs shlyapoff_app`). Вы можете переопределить их через переменные `ADMIN_USERnAME` и `ADMIN_PASSWORD` в файле `.env`.
